# <a name="c6211"></a>C6211
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

警告 C6211:正在泄漏内存\<指针 > 由于出现异常。 请考虑使用局部 catch 块清理内存  
  
 此警告意味着，分配的内存不被释放时引发异常。 位于路径末尾的语句可能会引发异常。 分析器将检查此条件时，才`_Analysis_mode_(_Analysis_local_leak_checks_)`指定 SAL 批注。 默认情况下，此批注指定的 Windows 内核模式 （驱动程序） 代码。 有关 SAL 注释的详细信息，请参阅[使用 SAL 注释减少 C /C++代码缺陷](../code-quality/using-sal-annotations-to-reduce-c-cpp-code-defects.md)。  
  
## <a name="example"></a>示例  
 下面的代码将生成此警告，因为可能在第二次分配时引发异常，从而泄漏第一个分配，或者，可能会在“`code ...`”注释所代表的代码中的某个位置引发异常，从而泄漏两个分配。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4   
#include <sal.h>  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
void f( )  
{  
    char *p1 = new char[10];  
    char *p2 = new char[10];  
  
    // code ...  
  
    delete[] p2;  
    delete[] p1;  
}  
  
```  
  
 若要使用相同方法并纠正此问题，请添加异常处理程序：  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <new>  
#include <iostream>  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f()  
{  
    char *p1 = nullptr;   
    char *p2 = nullptr;  
  
    try  
    {  
        p1 = new char[10];  
        p2 = new char[10];  
  
        // code ...  
  
        delete [] p2;  
        delete [] p1;  
    }  
    catch (const bad_alloc& ba)  
    {  
        cout << ba.what() << endl;  
        delete [] p2;  
        delete [] p1;  
    }  
    // code ...  
}  
  
```  
  
 若要完全避免这些潜在的泄露问题，请使用 C++ 标准模板库 (STL) 提供的机制。 其中包括[shared_ptr](https://msdn.microsoft.com/library/1469fc51-c658-43f1-886c-f4530dd84860)， [unique_ptr](https://msdn.microsoft.com/library/acdf046b-831e-4a4a-83aa-6d4ee467db9a)，并[向量](https://msdn.microsoft.com/library/c1431ad8-c0b6-4dbb-89c4-5f651e432d7f)。 有关详细信息，请参阅[智能指针](https://msdn.microsoft.com/library/909ef870-904c-49b6-b8cd-e9d0b7dc9435)并[C++标准库](https://msdn.microsoft.com/library/a37d3ba3-58af-47c7-9ee2-441ccd7b77ee)。  
  
```cpp  
// cl.exe /analyze /c /EHsc /nologo /W4  
#include <sal.h>  
#include <vector>  
#include <memory>  
  
using namespace std;  
  
_Analysis_mode_(_Analysis_local_leak_checks_)   
  
void f( )  
{  
    // use 10-element vectors in place of char[10]  
    vector<char> v1;  
    vector<char> v2;  
  
    for (int i=0; i<10; ++i) {  
        v1.push_back('a');  
        v2.push_back('b');  
    }  
    // code ...  
  
    // use unique_ptr if you still want char[10]  
    unique_ptr<char[]> a1(new char[10]);  
    unique_ptr<char[]> a2(new char[10]);  
  
    // code ...  
  
    // No need for delete; vector and unique_ptr   
    // clean up when out of scope.  
}  
  
```
